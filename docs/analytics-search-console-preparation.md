# Analytics・Search Console活用準備記録

Version 1.0  
最終更新：2026-07-16

## 目的

既存43ツールのブランド改善完了後に、推測ではなく利用状況と検索状況をもとに改善優先度を判断できるよう、現在の計測・インデックス実装、取得可能な指標、未確認事項、判断ルールを整理する。

本書は計測結果そのものではなく、データ活用を始めるための準備記録である。測定ID、所有権確認トークン、アカウント情報、利用者の入力内容は記載しない。

## 現在地

- 公開中43ツール、6カテゴリの個別ブランド改善、カテゴリ横断レビュー、全ツール最終横断レビューは完了している。
- 最終レビュー時点で重大・中項目は残っていない。
- ブランド育成フェーズは継続する。
- Analytics・Search Consoleの実測データを確認する前に、ホーム掲載順、関連ツール、SEO設定、カテゴリ分類を推測で変更しない。
- 運営基盤整備はロードマップのPhase 3に維持し、今回、自動取得・ダッシュボード・イベント計測は実装しない。

## Analyticsの導入状態

### 計測方式

- Google Analytics 4の`gtag.js`を各HTMLへ直接埋め込んでいる。
- Google Tag Managerのコンテナコードは確認されなかった。
- sitemap掲載60ページすべてで、同一の測定先に対するスクリプト読み込みと`config`呼び出しが各1回存在する。
- 公開ツール43ページ、ホーム、全ツール一覧、ゴマ知識、固定ページを含む。
- sitemap対象外のゴマ知識テンプレートにも計測コードがある。テンプレートは`noindex, nofollow`だが、直接開いた場合はページビューへ混入する可能性がある。
- 所有権確認用HTMLファイルには計測コードがない。

### ページビュー

- 静的な複数ページ構成であり、各ページの`config`呼び出しによる標準ページビュー計測を利用している。
- 仮想ページビュー処理、`send_page_view: false`、追加のページビュー送信は確認されなかった。
- 同一ページでの計測コード重複読み込みは確認されなかった。
- ホスト名による本番・開発環境の分離処理はない。ローカルサーバーなどでページを開き、計測スクリプトが読み込まれた場合は開発時アクセスが混入する可能性がある。

### イベント

- リポジトリ内に`gtag('event', ...)`による独自イベントは確認されなかった。
- ツール実行、再実行、コピー、保存、関連ツール遷移、ゴマ知識遷移、エラーは、現在のコードでは独自イベントとして送信していない。
- GA4管理画面側の拡張計測設定はコードから確認できない。
- 新規イベントを追加する場合も、入力値・生成結果・計算式・エラー内の入力値をパラメータへ含めない。

### Cookie・同意管理

- 計測コードはページ読み込み時に実行される。
- Cookie同意バナー、同意状態の保存、Consent Mode、同意前の計測抑止は確認されなかった。
- プライバシーポリシーにはGoogle AnalyticsとCookieによるアクセス情報収集、ブラウザ設定によるCookie無効化が記載されている。
- 実装と説明の存在は確認したが、地域別法令への適合性は本レビューでは判断しない。必要に応じて専門家確認の対象とする。

## 入力データの取り扱い

コードから確認できた事実は次のとおり。

- 入力値を送信する`fetch`、`XMLHttpRequest`、`sendBeacon`、WebSocketは確認されなかった。
- 独自Analyticsイベントはなく、入力文章、比較文章、パスワード、QRコード内容、健康・学習・給与情報、計算式、計算結果をイベントパラメータへ渡す処理は確認されなかった。
- 入力内容をURLクエリ、URLハッシュ、ページタイトルへ反映する処理は確認されなかった。
- `localStorage`は単位変換のカテゴリ選択保持にのみ使用されている。入力値の保存は確認されなかった。
- `sessionStorage`、Cookieへのツール入力値保存、入力値を含む独自エラーログは確認されなかった。
- QRコード生成は外部配信のライブラリを読み込むが、リポジトリ内の生成処理はブラウザ内で行われ、入力値を外部へ送信する処理は確認されなかった。
- GA4管理画面側の拡張計測、データ保持、内部トラフィック除外は管理画面で要確認。

## Search Console・公開構造

### 所有権確認

- リポジトリ直下にGoogle Search Console所有権確認用のHTMLファイルがある。
- GitHub Pages設定では内部運営資料だけを除外しており、確認用ファイルを除外する設定はない。
- コード上で確認できる所有権確認方法はHTMLファイル方式である。
- DNS、Google Analytics、Google Tag Managerなど他方式の併用状況と、現在の所有権確認成功状態は管理画面で要確認。
- 確認用ファイルは削除・改名しない。

### sitemap

- ルート直下に`sitemap.xml`があり、HTTPS・非wwwの正式URLを60件掲載している。
- 内訳は公開ツール43件、ゴマ知識10件、ホーム・一覧・固定ページ7件。
- URL重複、存在しないファイル、公開ツールの掲載漏れは確認されなかった。
- sitemap掲載60ページのcanonicalは、各sitemap URLと一致している。
- 所有権確認ファイルと`noindex`のゴマ知識テンプレートはsitemap対象外である。
- `lastmod`は2026年7月10日以前で、直近のブランド改善による公開ページ更新を反映していない。URLやインデックス可否の不備ではないが、次回のsitemap更新時に運用ルールを確認する。

### robots・noindex・canonical

- `robots.txt`は全クロールを許可し、正式なsitemap URLを指定している。
- sitemap掲載ページに`noindex`は確認されなかった。
- `noindex, nofollow`はsitemap対象外のゴマ知識テンプレートだけで確認された。
- sitemap掲載60ページのcanonical不一致は確認されなかった。
- HTTPステータス、リダイレクト、実際のクロール・インデックス状態は公開HTTP応答またはSearch Console管理画面で要確認。

## 改善判断ルール

### 今すぐ対応

- リンク切れ、noindex・canonical・sitemapの明確な設定ミス
- ページビューの重複送信
- 入力値・生成結果の計測混入
- モバイルで主要操作ができない問題
- 検索結果の説明とページ内容の明確な不一致

### データ蓄積後に判断

- ホーム掲載順と重複導線
- 関連ツールの順番・組み合わせ
- title、meta description、リード文
- ゴマ知識の追加優先度
- 小規模UI改善、新規ツール候補

### 運営基盤整備

- イベント命名規則と計測仕様書
- 内部トラフィック除外・テスト手順
- 定期レポート、データ保持方針
- 自動検査、自動取得、ダッシュボード、ツール管理台帳

これらはロードマップのPhase 3で扱い、今回実装しない。

### 放置可能

- 利用者影響がなく、実データでも問題を確認できない仮説
- ブランド方針と無関係な軽微な見た目の差
- 改善効果に対して工数が大きい変更

### Search Console

- 直近7日・28日・3か月と前期間の比較

## 定期確認案

- 毎週：計測停止・急減、インデックス問題、主要ページの異常、手動対策・セキュリティ問題
- 毎月：ツール別表示、検索クエリ、CTR、順位、モバイル比率、流入元、改善候補の優先順位
- 四半期：カテゴリ傾向、ホーム構成、ゴマ知識・新規ツール候補、ロードマップ・運営負荷

正式な頻度は、初回データ量と運営負荷を確認して決定する。
