# ゴマ早見表ロードマップ Version1.1

最終更新：2026-07-22

> このドキュメントはゴマツール開発・運営専用の内部資料です。
>
> 利用者向けの公開資料ではありません。

## 目的

ゴマ早見表シリーズとして追加を検討する候補と、その検討順序を管理する。

思いつきで追加せず、ブランド適合性、検索需要、保守性、利用者価値を総合的に評価し、十分な価値を確認できた候補だけを個別仕様化・実装する。

ゴマ早見表のブランド方針は[`docs/goma-reference-table-spec.md`](../goma-reference-table-spec.md)、構成・UI・導線の実装ルールは[`docs/goma-reference-table-implementation-spec.md`](../goma-reference-table-implementation-spec.md)に従う。

---

## この文書の役割

- 本書：候補、検討順序、評価結果、採用・保留状況を管理する
- ゴマ早見表ブランド仕様書：シリーズの役割とブランド方針を管理する
- ゴマ早見表共通実装仕様書：すべての早見表へ適用する実装・確認基準を管理する
- 運営ロードマップ：ゴマツール全体の開発フェーズと優先順位を管理する
- 個別仕様書：採用した早見表固有のデータ、URL、SEO、更新方法を定義する

本書への掲載は、公開や実装の決定を意味しない。

## 候補管理表

候補を個別仕様化する前に、次の項目を評価する。未調査の項目は推測で埋めない。

| 候補 | 検索需要 | 保守性 | ブランド適合性 | ゴマ知識展開 | 関連ツール連携 | 総合優先度 | 状態 | 備考 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  | S〜C | 未評価 |  |

評価は原則として5段階で記録する。

- 検索需要：高いほど星を多くする
- 保守性：更新負荷が低く、正確性を維持しやすいほど星を多くする
- ブランド適合性：「毎日に、一歩便利。」を体現するほど星を多くする
- ゴマ知識展開：公開済み・企画可能な知識との自然な連携が強いほど星を多くする
- 関連ツール連携：利用者の次の行動につながる実在ツールが多いほど星を多くする
- 総合優先度：評価結果とシリーズ順を踏まえ、S・A・B・Cで判断する

状態は「未評価」「調査中」「採用」「保留」「見送り」「公開済み」のいずれかを使用する。

---

## 採用基準

次の条件を満たす候補を優先する。

- 検索需要または明確な利用者需要が確認できる
- 長期的な保守負荷が低い
- スマートフォンで一覧性を保てる
- 入力せず利用できる
- ブランドコンセプト「毎日に、一歩便利。」と一致する
- ゴマ知識へ自然につながる
- 関連ツールとの回遊をつくれる
- 信頼できる情報源と更新方法を確保できる
- 既存の計算ツール、入力不要ツール、ゴマ知識と役割が重複しない

---

## 保留基準

次に該当する候補は、保守方法や提供範囲を確定できるまで保留する。

- 法改正が多い
- 料金改定が多い
- 外部APIへ依存する
- 自治体、地域、契約による差異が大きい
- 頻繁なメンテナンスが必要になる
- 信頼できる情報源を継続して確認できない
- スマートフォンで一覧性を保てない

---

## 採用後の進め方

1. 候補管理表で個別評価する
2. 採用・保留・見送りを判断する
3. 採用候補の情報源と更新方法を確認する
4. 個別仕様書を作成する
5. 実装・レビュー・動作確認を行う
6. 公開後、本書の状態を「公開済み」へ更新する

URL、SEO設定、データ内容、公開順は個別仕様書で確定する。候補名だけを根拠に実装へ進まない。

---

## 更新ルール

次の場合に更新する。

- 新しい候補を正式な検討対象へ追加した
- 候補の評価を完了した
- 採用、保留、見送りを判断した
- 個別仕様書の作成を開始した
- 早見表を公開した
- 法改正や情報更新により優先度を見直した

軽微な文章修正や実装途中の細かな変更は、Gitのコミット履歴で管理する。

---

## 運用方針

新しい候補は本ロードマップへ追加し、十分な価値を確認できたものだけを仕様化・実装する。

思いつきだけでは実装せず、ブランド全体の統一性と継続的な正確性を維持する。
