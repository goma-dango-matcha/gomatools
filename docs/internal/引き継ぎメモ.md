- サービス名: ゴマツール

- サービス説明: 無料で使える便利なツールと、日常に役立つ知識を提供するWebサービス。

## 8. ブランド改善 共通方針

改善時の基本内容:

- 主要情報を最優先する
- 重複文章を整理する
- 注意事項や説明を補足情報へ集約する
- 🍵 ゴマワンポイントを日常で役立つ内容へ改善する
- 公開済みのゴマ知識へ自然な導線を設ける
- 未公開記事は公開画面へ表示しない
- 利用シーンが近い関連ツールを2〜5件表示する
- スマートフォンの改行、行間、余白、スクロール量を確認する
- カテゴリ完了後はブランド改善レビューを実施する
- 共通コンポーネントのHTML構造・CSSクラス・表示順・デザインを横断確認する
- レビュー完了後に引き継ぎメモを更新する

原則として変更しない内容:

- 既存機能
- 計算ロジック
- 日付処理
- SEO設定
- URL
- 既存の構造化データ

## 12. ゴマ知識

ゴマ知識はSEOだけを目的とした記事ではない。

ツールの結果や仕組みを初心者にも分かりやすく説明し、利用者の理解を深めるコンテンツとして育成する。ツールと記事を相互に結び、必要な人が自然に詳しい情報へ進める構成を目指す。

公開済み記事のリンクは実ファイルの存在を確認する。未公開記事へのリンクは公開画面へ出さない。

## 14. GitHub運用

- 作業対象はGitHub Desktopで管理している正規リポジトリ。
- コピー作業は行わず、管理フォルダを直接編集する。
- Codexは実装・検証まで行い、原則としてCommit・Pushは行わない。
- GitHub Desktopで差分を確認してからCommitし、その後Pushする。
- 変更と無関係な既存差分を上書きしない。
- 通常のGitが使えない場合はGitHub Desktop同梱Gitを使用し、`safe.directory`を指定する。
- 関連リンクはディレクトリ基準の相対パスを優先する。
- docsへ現行仕様を追加・更新した場合は `docs/spec-index.md` も同期する。
- 文書整理だけの場合、公開ページの最終更新月は変更しない。

## 15. SEO状況

リポジトリで確認できる現在の状況:

- Google Search Console所有権確認用ファイルを設置済み
- `robots.txt`を設置済み
- `sitemap.xml`を設置済み
- `robots.txt`からsitemapを案内済み
- 各主要ページにtitle、meta description、canonical、OGP、構造化データを順次整備

Google側の現在の状況:

- Search Console上のインデックス登録数、除外理由、検索クエリ、掲載順位はリポジトリから確認できないため、管理画面で要確認。
- 公開から日が浅いため、短期間の数値だけで大きな方針変更を行わない。

## 16. SEO運用

- Search Consoleでサイトマップの取得状況を確認する。
- 新規公開ページや重要な更新ページはURL検査を行う。
- URL検査で問題がある場合はライブテストを行う。
- 公開状態とクロール可否を確認後、必要に応じてインデックス登録をリクエストする。
- インデックス未登録の場合は、noindex、robots、canonical、リンク導線、内容重複を確認する。
- 一時的な未登録や順位変動だけを理由に、URLやSEO設定を頻繁に変更しない。

既存の運用ルール・プロジェクト情報・注意事項・過去の管理情報は削除しない。

- 軽微なUI修正
- 誤字脱字修正

- レイアウト微調整

## 運用方針

このファイルを、ゴマツール開発・運営におけるプロジェクト状況の唯一の最新情報（Single Source of Truth）として扱う。

新しいChatGPTチャットを開始する際は、本ファイルを最初に共有する。プロジェクト状況が変わった場合のみ更新し、日常的な小規模修正では更新しない。

公開サイト用ドキュメントとは役割を分離し、内部運営資料として長期的に管理する。GitHub Pagesの生成対象からは `docs/internal` を除外し、公開サイト内・sitemap・利用者向けページからリンクしない。
