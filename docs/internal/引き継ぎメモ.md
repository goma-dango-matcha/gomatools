## 1. サービス概要

- サービス名: ゴマツール
- ブランドコンセプト: **毎日に、一歩便利。**
- サービス説明: 無料で使える便利なツールと、日常に役立つ知識を提供するWebサービス。
- ブランド育成: 単発のツール集ではなく、使いやすさ・分かりやすさ・安心感を備えた一貫性のあるサービスブランドとして育成している。
- スマホファースト: 小さな画面でも主要情報へすぐ到達でき、迷わず操作できる構成を優先する。
- タイパ重視: 短時間で答えが分かり、必要な場合だけ補足情報や関連知識へ進める体験を重視する。
- AI・Googleを意識した設計: 人にも検索エンジンにも内容と役割が伝わる見出し、文章、構造を整備する。
- 目指す方向: 利用者が毎日の疑問を迷わず解決でき、必要に応じて関連ツールやゴマ知識へ自然に移動できるサービスを目指す。

## 8. ブランド改善 共通方針

- 主要情報を最優先する
- 重複文章を整理する
- 注意事項や説明を補足情報へ集約する

- 利用シーンが近い関連ツールを2〜5件表示する

## 12. ゴマ知識

公開済み記事のリンクは実ファイルの存在を確認する。未公開記事へのリンクは公開画面へ出さない。

## 14. GitHub運用

- 作業対象はGitHub Desktopで管理している正規リポジトリ。
- コピー作業は行わず、管理フォルダを直接編集する。
- Codexは実装・検証まで行い、原則としてCommit・Pushは行わない。

- 変更と無関係な既存差分を上書きしない。
- 通常のGitが使えない場合はGitHub Desktop同梱Gitを使用し、`safe.directory`を指定する。
- 関連リンクはディレクトリ基準の相対パスを優先する。
- docsへ現行仕様を追加・更新した場合は `docs/spec-index.md` も同期する。
- 文書整理だけの場合、公開ページの最終更新月は変更しない。

## 15. SEO状況

サイト側の確認状況:

- `robots.txt`: 全クロールを許可し、sitemap URLを指定
- `sitemap.xml`: 公開60URLを掲載。公開43ツールの漏れ・重複なし
- canonical: sitemap掲載60ページでURLと一致
- noindex: sitemap対象外のゴマ知識テンプレートだけに設定
- GitHub Pages: 正常
- Google Search Console所有権確認用ファイル: 設置済み
- 主要ページのtitle、meta description、OGP、構造化データ: 順次整備済み

Search Console管理画面の前回確認状況（今回の作業では再確認できていない）:

- サイトマップは「取得できませんでした」が継続している。
- URL検査のライブテストでは「URL は Google に登録できます」を確認済み。
- ページはインデックス可能な状態であることを確認済み。
- インデックス登録リクエストはGoogle側のエラーで完了できていない。

コード上はHTMLファイル方式の所有権確認、robots、sitemap、canonical、noindexの整合を確認済み。sitemapの`lastmod`は直近のブランド改善を反映していないため、次回更新時に運用ルールを確認する。公開から日が浅いため、短期間の数値だけで大きな方針変更を行わない。

## 16. SEO運用

- Search Consoleは週1回を目安に確認する。
- URL検査は新規公開ページや重要ページを中心に行う。
- URL検査で状態を確認し、必要に応じてライブテストを行う。
- ページがインデックス可能であることを確認後、必要に応じてインデックス登録をリクエストする。
- サイトマップの再送信は大規模更新時を基本とし、頻繁に繰り返さない。
- インデックス未登録の場合は、noindex、robots、canonical、リンク導線、内容重複を確認する。
- 一時的な未登録や順位変動だけを理由に、URLやSEO設定を頻繁に変更しない。

- `gtag.js`を直接埋め込み、sitemap掲載60ページで各1回読み込んでいる。Google Tag Managerと独自イベントは未使用。
- ツール入力値・生成結果・計算式をAnalyticsイベントへ送る処理は確認されていない。
- Cookie同意バナー、Consent Mode、開発環境の除外処理は確認されていない。
- 基本確認頻度: 週1回を目安とし、新規公開や大きな改善の後にも確認する。
- 主に見る項目: 利用者数、人気ページ、流入経路。
- 必要に応じてセッション、表示回数、ランディングページ、エンゲージメント、利用端末も確認する。

## 更新方針

既存の引き継ぎメモをベースとし、既存内容は原則として維持する。

今回のブランド改善で変更が必要な項目のみ更新・追記する。

既存の運用ルール・プロジェクト情報・注意事項・過去の管理情報は削除しない。

## 更新ルール

次の場合のみ更新する。

- 新しいツールを公開した
- ブランド改善が1カテゴリ完了した
- 開発フェーズが変更された
- ブランド方針が変更された
- 共通仕様が追加・変更された
- SEO運用方針が変更された
- Analytics運用方針が変更された
- 今後やることが大きく変更された

## 更新しない内容

次の変更では、この引き継ぎメモを更新しない。

- 軽微なUI修正
- 誤字脱字修正
- CSS調整
- レイアウト微調整
- ボタン位置変更
- アイコン変更
- 軽微な文章修正
- 個別ツールの小規模改善
- バグ修正のみ
- コミットのみの変更

これらはGitHubのコミット履歴で管理する。

## 運用方針

このファイルを、ゴマツール開発・運営におけるプロジェクト状況の唯一の最新情報（Single Source of Truth）として扱う。

新しいChatGPTチャットを開始する際は、本ファイルを最初に共有する。プロジェクト状況が変わった場合のみ更新し、日常的な小規模修正では更新しない。

公開サイト用ドキュメントとは役割を分離し、内部運営資料として長期的に管理する。GitHub Pagesの生成対象からは `docs/internal` を除外し、公開サイト内・sitemap・利用者向けページからリンクしない。
